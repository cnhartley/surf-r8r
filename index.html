Introduction

This project provides a foundation for a larger project which may not be fully completed by the end of the Spring Quarter 2014 in CSC 481 - Knowledge-Based Systems at California Polytechnic State university in San Luis Obispo. This project entails user-based preference models of data collected from various sources related to oceanic conditions. These models represent the surfing conditions over a period of time that can be rated by the user.

Project Overview

The large project encompuses a client-server web application system. This system contains the front-end user interface, web-based application, and the server-side software. The user interface implementation utilizes HTML5 and CSS3 with JavaScript to support any dynamic content. While the back-end consists of the web-server generating the user interface pages, a database for all of the collected models and user information, and the software to mine the data from the require resource websites. The data miner will also compile the collected information into the appropriate models to store within the database. Finally, additional program(s) to monitor the data model collection in the database and watch for models that users may be interested in.

Background

Currently, there are not any application that take in user rating on surfing conditions in order to build a recommendation-like model. There are several websites and services that inform people of current ocean condition and some allow users to set alerts on very small sets of specific data pieces. And there are plenty of weather websites displaying the current and predicted conditions. But none that put it all together and allow the user to rate the data set as a whole. This ability enables custom user models of their own important data and hopefully provides meaningful alert notifications of conditions that they may really enjoy.

Main Features

For the end user of this application concept, this project entails the detection of their hand gestures and mapping them to interact with their computer system. These primary features include:

Raw Real-Time Data Collection
Ability to collect the real-time data from the various sources and compile them into the data models to represent the ocean at a given time.
Raw Prediction Data Collection
Collecting future weather forecast data from various sources and compile them into the data models to represent the future ocean conditions at a given time.
User Login to the Application
Enables the user to have their own custom preference models based on their previous ratings of ocean condition models gathered.
User Indicates a Rating for a Particular Surf Session
Provides the ability for a logged in user to rate a particular time frame of collected ocean models.
User Rating Matched to a Set of Raw Data Collected to Create a User Preference Model
Based on the logged in users rating, refines their own custom preference model based on the now rated ocean models.
